## Errores y lecciones técnicas

### 1. `filter: blur()` crea un containing block que clipea overflow
**Problema**: `MarketIntelligence` usa `-mx-4 md:-mx-6` (márgenes negativos) para expandirse. Dentro de un contenedor con `filter: blur()`, CSS crea un nuevo containing block que clipea todo lo que se sale → las cards desaparecían.

**Intentos fallidos**:
- `overflow-visible` en el contenedor blur → NO funciona, `filter` siempre crea containing block
- Revertir el patrón título-fuera-blur → incorrecto, el patrón era correcto

**Solución**: Quitar los márgenes negativos del componente cuando está dentro del blur (`hideHeader` prop condiciona el className).

**Lección**: `filter` en CSS (blur, drop-shadow, etc.) SIEMPRE crea un nuevo stacking context y containing block. No hay workaround con overflow. Los hijos con márgenes negativos se clipean. Si un componente usa márgenes negativos, necesita una variante sin ellos para funcionar dentro de `filter`.

### 2. No revertir patrones que funcionan
**Problema**: El patrón "título como title prop, contenido como children" funcionaba en todas las secciones. Cuando en Inteligencia de Mercado el contenido se veía vacío, revertí el patrón 3 veces antes de investigar la causa raíz.

**Lección**: Si un patrón funciona en 5 lugares y falla en 1, el problema no es el patrón — es algo específico de ese componente (en este caso, márgenes negativos). Diagnosticar antes de revertir.

### 3. `AuthGatedSection` vs `AuthGateServer`
- `AuthGateServer`: blurrea todo sin distinción — títulos incluidos
- `AuthGatedSection`: acepta `title` prop que queda fuera del blur
- Para secciones con heading interno (ej: `MarketIntelligence`), necesitas un prop `hideHeader` para evitar duplicados

### 4. Blur level por página
- 3px funciona bien para home (textos medianos)
- 5px necesario para zona (precios grandes como $24,500/m² se leían con 3px)
- `AuthGatedSection` acepta `blur` prop para control granular

### 5. Componentes con estado propio dentro de blur
- `ComparadorClient` tiene su hero dinámico dentro del client component
- No se puede separar hero del content a nivel server
- Solución: usar `AuthGateInline` (client-side) dentro del componente, envolviendo solo el contenido post-hero

### 6. Stitch timeouts
- `generate_screen_from_text` con GEMINI_3_1_PRO timeout frecuente
- GEMINI_3_FLASH también puede dar timeout
- Workaround: esperar 30s y checar con `list_screens` — la generación suele completarse server-side
